# Café Management System / نظام إدارة المقهى (POS)

نظام متكامل واحترافي لإدارة المقاهي ونقاط البيع (POS). تم بناء المشروع باستخدام إطار عمل **Flutter** ليعمل بكفاءة عالية على أنظمة تشغيل سطح المكتب مثل **ويندوز (Windows)**، مع قاعدة بيانات محلية مرنة لتسهيل التخزين والمعالجات السريعة بدون الحاجة لإنترنت. الأداة مصممة بأسلوب هندسة معمارية نظيفة لضمان قوة الأداء وسهولة التوسع والصيانة لاحقاً.

---

## التقنيات المستخدمة (Tech Stack)

* **إطار العمل الأساسي**: Flutter 3.x
* **لغة البرمجة**: Dart
* **إدارة الحالة (State Management)**: `flutter_bloc` و `Cubit`.
* **قاعدة البيانات**: `sqflite` للتعامل مع قاعدة بيانات SQLite مع استخدام `sqflite_common_ffi` لدعم بيئة أوامر الويندوز.
* **البنية التحتية وحقن الاعتمادية (DI)**: حزمة `get_it` لربط مزودي البيانات (Repositories) وصناديق التحكم (Cubits) ببعضها.
* **إدارة المسارات (Routing)**: حزمة `go_router` مع نظام الحراسة (Auth Guards) والتوجيه الداخلي (ShellRoute) لضمان بنية جانبية مستقرة للحواسيب.
* **تصدير التقارير**: حزم `pdf` و `printing` لإنشاء فواتير وملفات PDF باللغات العربية والإنجليزية وتصديرها أو طباعتها.
* **الرسوم البيانية (Charts)**: حزمة `fl_chart` لإنشاء رسوم بيانية مالية تفاعلية.
* **اللغات (Localization)**: دعم محلي عبر تقنية `flutter_localizations` المدمجة.
* **التشفير (Security)**: حزمة `crypto` لتشفير كلمات مرور التطبيق المحتفظة محليًا برمز تجزئة (SHA-256).

## الخصائص والميزات الأساسية (Core Features)

### 1- نظام تسجيل الدخول والمصادقة الموثوق
* يعتمد النظام على كلمة مرور موحدة تُعين من قِبل مدير المقهى عند فتح البرنامج لأول مرة.
* لا توجد بيانات مخزنة كسلسلة نصية عادية؛ بل تحفظ باستخدام تشفير SHA-256 لضمان أمان الإيرادات ومحتويات النظام.

### 2- نظام نقطة بيع متقدم (POS - Billing Screen)
* واجهة مقسمة (Grid & Sidebar) حيث يتم سرد المنتجات وتصفيتها بضغطة زر حسب الفئة.
* **الفواتير المتعددة (Multi-tab System)**: يمكن فتح عدة فواتير بالتوازي لخدمة عدة طاولات/عملاء في وقت واحد بدون ضياع بيانات الفاتورة المفتوحة.
* حساب تلقائي ولحظي للإجمالي والضرائب وتحديد طريقة الدفع (Cash \ Card).

### 3- لوحة التحكم والتحليلات (Dashboard + Analytics)
* واجهة تستعرض ملخص الإيرادات النقدية والبنكية مقارنة بالمصاريف لحساب "صافي الربح".
* رسم بياني يُظهر مستوى ومعدل المبيعات على مدار الأيام باستخدام بطاقات استعراض احترافية Material 3.
* عرض لأكثر المنتجات مبيعًا من خلال مؤشرات تقييم.

### 4- إدارة المنتجات (Products Management)
* توفير نظام لإدخال فئات المنتجات (Categories) والمنتجات المندرجة تحتها.
* تسجيل صور للمنتجات ومعرفة كم بيع منها لمتابعة الإقبال.

### 5- إدارة المصروفات (Expenses)
* التحكم التام وتتبع المبالغ الخارجة لغرض دفع فواتير، شراء مستلزمات، صيانة، إلخ.
* خصم المصروفات فوراً من الدخل الصافي للمقهى لإعطاء إحصائية صادقة للمالك.

### 6- واجهة تقارير قابلة للطباعة (PDF Reports)
* إنشاء تقارير مبيعات مفصلة يمكن استخراجها بضغطة زر من صفحة "لوحة التحكم" مباشرة وطباعتها بصيغة (PDF). تقارير مرتبة تتضمن ترويسة تحمل بيانات المقهى و جداول لمبيعات أطول الفترات.

### 7- التخصيص الكامل (Settings & Theming)
* **دعم الوضع المظلم (Dark Mode)** والوضع الفاتح، حيث يعتمد كلاهما على تصميم "Material 3" مع ألوان المقهى الهادئة والمميزة.
* **تعدد اللغات (Localization)**: واجهة مرنة تدعم التحويل لدعم اللغة الإنجليزية، العربية (RTL)، والبنغالية باستخدام خطوط مخصصة وواضحة جداً لكل لغة (مثل خط Cairo للعربية).

## الهندسة المعمارية (Architecture)

يعتمد التطبيق على معمارية **Clean Architecture** لضمان فصل المهام بوضوح كما يلي:

* **Presentation Layer**: تتضمن واجهات المستخدم (`screens`) ومكونات التطبيق (`widgets`) إضافة إلى الـ (`Cubit`/`State`) المسئول عن معالجة التفاعلات وعكس حالة البيانات.
* **Domain Layer (Repositories)**: تحتوي على واجهات ومنطق جمع البيانات الوسيط، مثل `InvoiceRepository`, `ProductRepository` للتحقق من شروط البيع والشراء.
* **Data Layer (Models & Database)**: تعريف هياكل الجداول، النماذج (Models)، وصنف الكائن الأساسي `DatabaseHelper` المسؤول الأول عن إرسال أوامر الـ SQL المعقدة وإنشاء الجداول وتوفير المعاملات المأمونة (Transactions).

## مخطط قاعدة البيانات (Database Schema)

يعتمد التطبيق على جداول مترابطة بالكامل:
* `auth`: يخزن كلمة مرور الإدارة بعد تشفيرها.
* `categories`: الفئات الرئيسية للمنيو (مشروبات ساخنة، عصائر، الخ).
* `products`: المنتجات الفرعية وترتبط برقم الفئة `category_id`.
* `invoices`: الفواتير المعالجة مع طرق الدفع والتاريخ والأوقات.
* `invoice_items`: تفاصيل كل مبيعة داخل الفاتورة، مع ربطها برقم المنتج والفاتورة.
* `expenses`: المصروفات المستقلة خارج عمليات المبيعات.

## بدء تشغيل النظام (Getting Started)

1. **المتطلبات الأساسية**: تأكد من أن بيئة عمل `Flutter` مهيئة لإنتاج تطبيقات الويندوز (Windows Desktop). يمكن تفعيلها برمجياً عبر:
   ```bash
   flutter config --enable-windows-desktop
   ```
2. **جلب الاعتمادات**: من داخل مجلد المشروع تأكد من تحميل الحزم الناقصة بواسطة الأمر:
   ```bash
   flutter pub get
   ```
3. **توليد اللغات (Localization)**: تشغيل أمر بناء اللغات لكي تتفعل مسارات (l10n).
   ```bash
   flutter gen-l10n
   ```
4. **تشغيل المشروع**: بادر بالتشغيل عبر اختيار نظام الويندوز. في واجهة الأوامر أكتب:
   ```bash
   flutter run -d windows
   ```

*ملاحظة: عند الفتح للمرة الأولى، سيوجهك النظام لإنشاء كلمة مرور (Master Password) أساسية؛ يُرجى الاحتفاظ بها جيداً لأنها الطريقة الوحيدة للدخول للمشروع وتعديل خصائصه.*
